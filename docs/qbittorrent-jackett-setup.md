# qBittorrent + Jackett Setup

Instructions for configuring qBittorrent web UI and Jackett search plugin — based on Windows setup, with Docker notes.

---

## qBittorrent Web UI

### Windows (current)

Config file: `%APPDATA%\qBittorrent\qBittorrent.ini`

```ini
[Preferences]
WebUI\Enabled=true
WebUI\Port=44011
WebUI\Address=*
WebUI\LocalHostAuth=false
WebUI\HostHeaderValidation=false
WebUI\AuthSubnetWhitelistEnabled=false
WebUI\CSRFProtection=true
WebUI\ClickjackingProtection=true
WebUI\SecureCookie=true
WebUI\Username=admin
WebUI\Password_PBKDF2=@ByteArray(d0ec493d27ed22320ed7846f37f7cb7c:8110294c441dca8054cc5c36b00017327be671af0b6c91ed8e59188b85eb0e03cbc02c02eb161c413e66fdb2ca49703ca55b4090dce302c0adbfc8a7f64f15a1)
```

**Important gotchas:**
- qBittorrent will refuse to start the web server if `Username` and `Password_PBKDF2` are not set, even if `LocalHostAuth=false`.
- The password value is PBKDF2-SHA512: `salt:hash` encoded as hex, prefixed with `@ByteArray(...)`.
- On Windows the file is `.ini`; on Linux it is `.conf` — same format, different extension.
- `HostHeaderValidation=false` is required when accessing via hostname or reverse proxy.

#### Generating a PBKDF2 password (PowerShell 5.1)

```powershell
$password = "your-password-here"
$saltBytes = New-Object byte[] 16
$rng = New-Object System.Security.Cryptography.RNGCryptoServiceProvider
$rng.GetBytes($saltBytes)
$salt = ($saltBytes | ForEach-Object { $_.ToString('x2') }) -join ''

$pbkdf2 = New-Object System.Security.Cryptography.Rfc2898DeriveBytes(
    $password,
    $saltBytes,
    100000,
    [System.Security.Cryptography.HashAlgorithmName]::SHA512
)
$hashBytes = $pbkdf2.GetBytes(64)
$hash = ($hashBytes | ForEach-Object { $_.ToString('x2') }) -join ''

Write-Host "@ByteArray($salt`:$hash)"
```

Paste the output as the value of `WebUI\Password_PBKDF2`.

### Docker

Use the [linuxserver/qbittorrent](https://hub.docker.com/r/linuxserver/qbittorrent) image.

Config file inside container: `/config/qBittorrent/qBittorrent.conf`

Mount a volume to `/config` and pre-populate `qBittorrent.conf` with the same settings as above (same key names, same `@ByteArray(...)` format).

```yaml
services:
  qbittorrent:
    image: lscr.io/linuxserver/qbittorrent:latest
    environment:
      - PUID=1000
      - PGID=1000
      - WEBUI_PORT=44011
    volumes:
      - ./config/qbittorrent:/config
      - ./downloads:/downloads
    ports:
      - 44011:44011
```

## Jackett

### Windows (current)

1. Download the latest Windows release from https://github.com/Jackett/Jackett/releases (pick `Jackett.Binaries.Windows.zip`).
2. Extract to `C:\m\Jackett\` (or any path without spaces).
3. Run `JackettTray.exe` or `jackett.exe` — it starts on port 9117 by default.

Config file: `C:\ProgramData\Jackett\ServerConfig.json`

```json
{
  "Port": 9117,
  "LocalBindAddress": "127.0.0.1",
  "AllowExternal": false,
  "APIKey": "<auto-generated on first run>"
}
```

The API key is auto-generated on first run. Read it from `ServerConfig.json` after starting Jackett.

### Docker

Use the [linuxserver/jackett](https://hub.docker.com/r/linuxserver/jackett) image.

Config file inside container: `/config/Jackett/ServerConfig.json`

```yaml
services:
  jackett:
    image: lscr.io/linuxserver/jackett:latest
    environment:
      - PUID=1000
      - PGID=1000
    volumes:
      - ./config/jackett:/config
    ports:
      - 9117:9117
```

## Jackett Search Plugin for qBittorrent

The plugin lets qBittorrent's built-in search use Jackett indexers.

### Plugin files

Download `jackett.py` from the official qBittorrent search-plugins repo:
```
https://raw.githubusercontent.com/qbittorrent/search-plugins/master/nova3/engines/jackett.py
```

Create `jackett.json` in the same directory:
```json
{
    "api_key": "<your Jackett API key>",
    "tracker_first": false,
    "url": "http://127.0.0.1:9117"
}
```

### Plugin directory locations

| Environment | Path |
|---|---|
| Windows | `%LOCALAPPDATA%\qBittorrent\nova3\engines\` |
| Linux / Docker qBittorrent | `~/.local/share/data/qBittorrent/nova3/engines/` |

**Windows gotcha:** The plugin must go in `%LOCALAPPDATA%` (i.e. `C:\Users\<user>\AppData\Local\qBittorrent\nova3\engines\`), NOT `%APPDATA%` — qBittorrent will not detect it from `%APPDATA%`.

### Verify plugin loaded

```
GET http://localhost:44011/api/v2/search/plugins
```

Should return a JSON array that includes an entry with `"name": "Jackett2"`.

## Enabling Jackett Indexers

### Via the Web UI

Go to `http://localhost:9117`, click **+ Add Indexer**, search for the tracker, click the wrench to configure credentials if needed, then **OK**.

### Via the API (bulk / programmatic)

The admin API requires a session cookie — the API key is only for torznab search, not the admin endpoints. Establish a session first, then GET the indexer config and POST it back to enable it.

**Flow per indexer:**
1. `GET /api/v2.0/indexers/{id}/config` — returns current config fields with defaults
2. `POST /api/v2.0/indexers/{id}/config` — save config (same JSON array POSTed back); returns `204` on success

**Bulk-enable all public indexers (PowerShell):**

```powershell
$session = New-Object Microsoft.PowerShell.Commands.WebRequestSession
Invoke-WebRequest -Uri "http://localhost:9117/UI/Dashboard" -WebSession $session -UseBasicParsing | Out-Null

$all = Invoke-RestMethod -Uri "http://localhost:9117/api/v2.0/indexers" -WebSession $session
$unconfigured = $all | Where-Object { $_.configured -eq $false -and $_.type -eq "public" }

$ok = 0; $fail = 0
foreach ($indexer in $unconfigured) {
    try {
        $config = Invoke-RestMethod -Uri "http://localhost:9117/api/v2.0/indexers/$($indexer.id)/config" -WebSession $session
        $r = Invoke-WebRequest -Uri "http://localhost:9117/api/v2.0/indexers/$($indexer.id)/config" `
            -Method POST -WebSession $session -ContentType "application/json" `
            -Body ($config | ConvertTo-Json -Depth 5) -UseBasicParsing -ErrorAction Stop
        if ($r.StatusCode -eq 204) { $ok++ } else { $fail++ }
    } catch { $fail++ }
}
Write-Output "Enabled: $ok  |  Failed: $fail"
```

**Notes:**
- Public indexers (~108 available) enable instantly — no network call to the tracker site.
- Semi-private and private indexers trigger a live login attempt to the site on every config POST, making bulk-enabling slow and mostly unsuccessful without credentials.
- Two public indexers (`divxtotal`, `postman`) require FlareSolverr (Cloudflare bypass) to configure — they will 500 without it.
- Indexer definitions are YAML files in `C:\m\Jackett\Definitions\`. Those with a `login:` section will make a network call during configuration.
